## 2. QA — проблемы верстки (по скриншоту)

### 2.1 🔴 Карточка обрезает текст без переноса

**Симптом:** В карточке видны оборванные строки — `"res"`, `"rs.github.io/docs/fa"`, `"gl"`, `"stdo"`, `"std"` — усечённые фрагменты About, GitHub URL и тела changelog.

**Причина:** `renderCard()` пишет строки без ограничения по ширине `cardW`. Viewport обрезает визуально, но не переносит. Длинные URL и слова обрезаются на полуслове.

**Затронуто:**
- About: `ui.DescStyle.Render(card.About)` — нет word-wrap
- GitHub URL: `"↗ https://"+t.GitHub` — всегда одна длинная строка
- Changelog body: `wrapText()` использует `len(word)` (байты, не руны) → ломается на Unicode

### 2.2 🔴 Хелп-панель обрезает текст справа

**Симптом:** Строки в хелп-панели уходят за правый край — `"Write the selected files to this file on"` и другие обрезаются.

**Причина:** `helpViewport` скроллит только вертикально. Строки шире `helpW` просто отрезаются. `--help` вывод многих инструментов бывает шире 80 символов.

**Решение:** Применять `wrapText(output, helpW-2)` при заполнении `helpCache`, до передачи в viewport.

### 2.3 🟡 Большой пустой блок в карточке

**Симптом:** Между Note/Tags и фрагментами текста — огромное пустое пространство.

**Причина:** Changelog блок пустой (`"Press [c] to load changelog"`), но занимает оставшееся пространство viewport фиксированной высоты. Карточка не сжимается по контенту.

### 2.4 🟡 Вертикальная черта `|` в строках хелп-панели

**Симптом:** Перед `--local-events` и `--remote-events` видны символы `|`.

**Причина (уточнить):** Либо это часть самого вывода `yazi --help`, либо правая граница `cardBox` (BorderRight: true) "наплывает" на хелп-текст из-за несоответствия ширины.

### 2.5 🟡 Левая панель не скроллится

**Симптом:** При 4 инструментах в списке видны только 2 (`nnn`, `bat`). Footer показывает `"4 tools"`.

**Причина:** Левая панель — просто `strings.Builder` без viewport. При маленьком терминале нижние инструменты выходят за границу рамки и обрезаются. Скролл не предусмотрен.

## 4. Технические проблемы

### 4.1 Race condition на `cache.json`

`Init()` запускает по горутине на каждый инструмент. Каждая вызывает `GetLatest` → `LoadCache` → мутирует локальную копию → `SaveCache` — без синхронизации. Последняя запись перетирает все предыдущие.

**Воспроизводится:** при 3+ инструментах с GitHub. После старта часть инструментов может показывать устаревший кэш.

### 4.2 Double-fetch при первом старте

`GetLatest` сохраняет `Languages: nil` (новый инструмент). `GetRepoCard` видит `Languages == nil` → cache miss → делает повторный запрос. Итого: 2 API-запроса вместо 1 при первом добавлении инструмента.
