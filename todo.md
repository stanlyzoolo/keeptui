### Изменение концепции проекта

Проект будет заключать в себе основную фичу - CLI/TUI Registry

#### Что это значит?

Внутри проекта добавляются/хранятся/архивируются CLI/TUI инструменты
для личных целей - фидбек, статусы и т.д. В проекте не хранятся горячие клавиши и команды.

#### Что значит "личные цели"?

Пользователь добавляет инструмент самостоятельно (указывая ссылку на репозиторий на github.com) - никаких yaml конфигов, парсинга `tldr` биндингов не храним и не ходим. 
за ними на github.com.
Если пользователю потребуются горячие клавиши или команды инструмента, из 
из проекта можно будет вызвать `<tool_name> --help`, либо `man <tool_name>`.  
Нужно предоставить пользователю возможность
выбрать что запустить, а именно на каждую команду своя клавиша в Help Bar (Строка подсказок).
Желательно раскрасивать вывод, отделив название команда и флагов от описания цветом.

#### Что нужно сделать?   
Будем полностью использовать пространство правой панели (Right Panel), разделив её дополнительно на две половины, которые
видны пользователю сразу (без табов):
- Убираем разделение на Keys и Commands
- Первая половина - карточка тулы. Отражаем в ней следующую информацию в указанном порядоке:
  + About - из репозитория на github
  + Stars - из репозитория на github
  + Releases (последнюю версию) - из репозитория на github
  + Languages (придумай как красиво отразить процентное соотношение) - из репозитория на github
  + Сhangelog - из репозитория на github (если доступен)  
  Нужно по смысловому наполнению разграничить блоки, используя horizontal divider.
- Вторая половина - output команд <tool_name> --help`, либо `man <tool_name>`
Левая панель:
- Убираем табы Hotkeys, оставив только My tools
