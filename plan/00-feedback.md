# Фидбек по todo.md

## Что сформулировано хорошо

- Чёткий концептуальный сдвиг: Registry вместо хранилища хоткеев
- Умное решение использовать `--help` и `man` — не нужно держать команды в YAML
- Двухпанельная правая панель без табов — правильный UX-выбор
- Разграничение блоков через horizontal divider — читаемо

---

## Пробелы и уточнения

### 1. `ToolMeta` нужно поле `GitHub`
В `meta.yaml` сейчас хранится только `name`, `status`, `added`, `tags`, `note`.  
Когда YAML-конфиги инструментов удаляются — GitHub URL некуда класть.  
Нужно добавить поле `github` в `ToolMeta` и обновить команду `keys track`.

**Текущая команда:**
```
keys track neovim --status trying
```
**Нужна новая форма:**
```
keys track neovim --github github.com/neovim/neovim --status trying
```

---

### 2. Languages — отдельный API-запрос
GitHub возвращает только основной язык (`language: "Go"`) в `/repos/{owner}/{repo}`.  
Для процентного соотношения нужен **отдельный запрос**:
```
GET /repos/{owner}/{repo}/languages
→ {"Go": 125000, "Shell": 4200, "Makefile": 800}
```
Это нужно учесть в кэше и в логике fetching.

---

### 3. ANSI-коды в выводе `--help`
Многие инструменты (bat, gh, docker) выводят `--help` с цветами через ANSI-escape коды.  
Нужно решить стратегию: стрипать или рендерить.  
Стрипать проще и надёжнее для TUI — иначе сырые escape-последовательности сломают отображение.

---

### 4. `man` может отсутствовать
На macOS многие CLI-инструменты не имеют man-страниц.  
Fallback: если `man <tool>` возвращает ошибку — показать сообщение "man page not available".

---

### 5. Структура `Tool` сливается с `ToolMeta`
Сейчас `Tool` (из YAML) и `ToolMeta` (из meta.yaml) — разные структуры.  
После удаления YAML-конфигов `Tool` теряет смысл как отдельная сущность.  
Единый источник правды — `ToolMeta` + GitHub API.

---

### 6. Changelog уже работает
`version.GetChangelog()` уже получает body последнего релиза с GitHub.  
Нужно только перенести вывод из попапа в правую панель (Tool Card).

---

### 7. Поиск ✓ Решено

**Левая панель** — `/` поиск по названиям инструментов остаётся.  
При совпадении правая панель автоматически показывает первый подходящий результат (Tool Card + Help Output).

**Правая панель** — отдельный поиск внутри вывода `--help`/`man`.  
Активируется отдельной клавишей (например `f` или `/` когда фокус на правой половине).  
Совпадения подсвечиваются, навигация по ним через `n` / `N`.
